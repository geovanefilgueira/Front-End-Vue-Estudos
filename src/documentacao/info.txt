Esta página traz desafios de alinhamento de componentes e este desafio reflete no
desenvolvimento de todo layout, onde os elementos de slogan e formulário estão encapsulados em uma div absolute"flutuante"
e estão alinhados com base nesta div flutuante, que ao redimensionar para um celular altera a propriedade absolute para 
relative deste modo os elementos internos se comportam de forma estruturada um abaixo do outro
ao se tornar responsivo as divs que contém o header imagem de bkg e filtro se torna absolute, não respeitando os 
demais elementos da página, assim a div que era flutuante assume o lugar do header e tanto o header e a ex div flutuante
 tomam o mesmo lugar um sobrepondo o outro


As demais divs ou section não tem nenhuma complicação pois o elementos não flutuam na tela


durante todo o desenvolvimento foi utilizado flexbox, o que pode se tornar muito repetitivo gerando código desnecessário
alternativas será a utilização de algum micro processador css, como saas, lessou um framework, 
ou até mesmo um framework próprio contendo o design systems do projeto onde cada elemento conter diversas classes
 assim como acontece no framework tailwind css
